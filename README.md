游戏策划文档：连接勇士（Web版卡牌游戏）

1. 游戏概述

1.1 游戏背景

《连接勇士》是一款基于浏览器的策略卡牌游戏，玩家扮演一名勇士，通过打出卡牌释放魔法和武器来对抗怪兽。游戏融合了卡牌构筑、职业系统和roguelike元素，强调策略性和连击组合。游戏设计为响应式网页，完美适配手机端操作。

1.2 游戏目标

- 主要目标：玩家在每一轮中击败三只怪兽即可获胜。
- 失败条件：玩家生命值归零。
- 长期目标：通过roguelike式的进程解锁新卡牌、职业和内容，提升游戏重玩价值。

1.3 核心特色

- 无费用系统：卡牌打出无资源限制，专注于连击和策略。
- 连接能量机制：通过打出同名卡积累能量，发动职业技能。
- 属性系统：卡牌可能带有火、冰、毒、电等属性，叠加层数增强效果。
- roguelike元素：随机事件、永久死亡（可选模式）和渐进式解锁。

2. 核心玩法

2.1 基本流程

- 玩家首先构筑一个30张卡的卡组（同名卡最多5张）。
- 每轮开始前，抽取5张卡作为手牌。
- 每回合分为玩家回合和怪兽回合：
   - 玩家回合：玩家打出手牌中的卡牌（只能打出同名卡，可多张一起打出），产生效果后卡牌返回卡组，回合结束抽卡补满5张手牌。
   - 怪兽回合：每只怪兽随机使用一个技能攻击玩家或增益自身。
- 击败当前轮次的所有三只怪兽后，玩家进入下一轮（难度提升）；玩家生命归零则游戏结束。
- 每轮游戏结束后根据怪兽的等级获得对应星点。

2.2 战斗机制

- 生命值与护盾：
   - 玩家和怪兽都有生命值（HP）和护盾（Shield），均有最大值。护盾可抵消物理攻击，魔法攻击直接扣除生命值。
   - 护盾优先扣除：物理攻击先扣护盾，护盾不足时再扣生命值；魔法攻击无视护盾。
   - 生命值和护盾不能超过最大值。
- 属性伤害：
   - 卡牌可能带有火、冰、毒、电等属性。属性伤害直接作用于生命值，并可叠加层数（每张卡增加一层）。
   - 属性效果示例：
      - 火：每层导致受到的伤害加深（受到伤害时，每1层火属性，伤害就增加1）。
      - 冰：每层导致发起的伤害减弱（发起攻击时，每1层冰属性降低1点造成的伤害。最多只能把降为0）。
      - 毒：每1层造成1点伤害（回合结束时结算），持续3个回合。
      - 雷：每1层减少2点护盾（回合结束时结算），持续3个回合。
- 效果对象：卡牌效果分单体和全体。打出多张同名卡时，效果合并计算。

3. 卡牌系统

3.1 卡牌类型

- 武器卡：物理攻击，先扣除目标护盾。示例：“铁剑”（单体物理伤害，无属性）、“火焰斧”（单体物理伤害+火属性）。
- 法术卡：魔法攻击，直接扣除生命值。示例：“火球术”（单体魔法伤害+火属性）、“寒冰风暴”（全体魔法伤害+冰属性）。
- 护盾卡：增加自身或目标的护盾。示例：“木盾”（增加单体护盾）、“元素屏障”（增加全体护盾）。
- 药水卡：恢复生命值或清除状态。示例：“治疗药水”（恢复单体生命）、“净化药水”（清除所有属性状态）。

3.2 卡牌属性

- 每张卡有名称、类型、效果描述、属性（可选）和层数（如果带属性）。
- 属性层数叠加：打出多张同名属性卡时，层数累加。
- 卡牌稀有度：普通（1星）、稀有（2星）、史诗（3星）（影响获取方式和强度），用于roguelike进度。

3.3 卡组构筑

- 卡组固定30张，同名卡最多5张。
- 玩家可在每轮游戏开始前修改卡组。
- 初始卡组为预设，随进程解锁新卡。

4. 战斗系统

4.1 回合流程

1. 玩家回合：
   - 打出手牌：选择一张同名卡（可多张同时打出），指定目标（单体/全体）。
   - 效果结算：计算伤害、护盾或状态变化。连接能量积累（打出n张同名卡，获得n-1点能量）。
   - 卡牌处理：使用后卡牌返回卡组底部。
   - 抽卡阶段：从卡组随机抽卡，补满5张手牌。
2. 怪兽回合：
   - 每只怪兽依次行动，随机使用一个技能（从1-6个技能中随机选择）。
   - 技能类型可能包括攻击（物理/魔法）、增益（护盾、恢复生命、清除状态）。
   - 效果结算后，回合结束。

4.2 伤害与状态计算

- 物理伤害公式：
"实际伤害 = 卡牌伤害 - 目标护盾"（护盾不足时，溢出伤害扣生命）。
- 魔法伤害公式：
"实际伤害 = 卡牌伤害"（直接扣生命）。
- 状态清除：药水卡可清除属性层数或效果。

5. 职业系统

5.1 职业概述

- 玩家可选择不同职业，每个职业有独特技能和连接能量上限。
- 初始职业示例：
   - 战士：高护盾和物理伤害，技能偏向防御和爆发。能量上限6。
   - 法师：高魔法伤害和属性叠加，技能偏向AOE。能量上限8。
   - 游侠：平衡型，技能侧重连击和敏捷。能量上限7。
- 更多职业通过星点兑换。
- 可以通过星点值提升该职业的生命值上限、护盾值上限或能量上限。

5.2 连接能量机制

- 积累方式：玩家回合打出同名卡时，积累连接能量。能量数 = 打出同名卡数量 - 1。
   - 示例：打出3张“火球术”，获得2点能量。
- 存储与使用：能量存储在连接卡槽中，上限由职业决定（默认6）。能量达到技能要求时可发动技能，技能发动消耗能量。
- 技能示例：
   - 战士技能“狂暴打击”（需3点能量）：本回合物理伤害翻倍。
   - 法师技能“元素爆发”（需5点能量）：本回合魔法伤害翻倍。


6. Roguelike元素

6.1 进程设计

- 随机生成：每轮怪兽组合、技能和事件随机生成，增加重玩性。
- 永久死亡：玩家失败后，卡组和进度重置（但可解锁永久内容如新职业）。
- 渐进式解锁：通过击败轮次获取的星点来兑换卡牌、职业。

6.2 事件系统

- 在轮次之间触发随机事件，例如：
   - 物理强化：本轮所有物理伤害+1。
   - 魔法强化：本轮所有物理伤害+1。
   - 护盾弱化：本轮开始时所有角色的护盾为0。
-事件有可能发生，也有可能不发生。
6.3 难度曲线

- 每轮难度递增：怪兽生命值、伤害提升，技能更复杂。

7. 用户界面设计（手机适配）

7.1 核心界面

- 主菜单：开始游戏、卡组编辑、卡片购买、玩家信息。
- 战斗界面：
   - 上方：本轮的信息（第几轮、第几回合、事件）。
   - 中央偏上：怪兽列表（名称、头像、生命值、护盾值、属性和层数）。
   - 下方偏下：玩家信息（名称、头像、生命值、护盾值、属性和层数）。
   - 侧边：连接能量槽和技能按钮。
   - 最下方：手牌区域（卡片以横向滑动列表展示，适应手机屏幕）。

8. 游戏流程

8.1 单局流程

1. 玩家选择职业和初始卡组。
2. 进入第一轮，抽取5张手牌。
3. 回合制战斗：玩家与怪兽交替行动，直到三只怪兽被击败或玩家死亡。
4. 获胜后进入下一轮（难度提升），失败则结束游戏。
5. 游戏结束后显示获得的星点值，星点值为击败的怪兽的合计等级。

8.2 宏观进度

- 通过多次游戏积累“星点”（货币），用于兑换新卡、解锁和升级职业。
